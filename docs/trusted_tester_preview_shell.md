# Trusted Tester Preview Shell

## Status

- Stage 12C.1.
- UI shell only.
- No deployment.
- No access gate.
- No feedback persistence.
- No AI KINTEL.

## Purpose

- Prepare a non-technical click path for a trusted external reviewer.
- Reduce need for CMD, repository, or developer explanation.
- Explain WATCHLIST, freshness, review, report preview, and feedback prompts.

## Preview Path

1. Start with radar overview.
2. Open candidate/project.
3. Check source freshness.
4. Read research snapshot.
5. Review report preview.
6. Leave structured feedback.
