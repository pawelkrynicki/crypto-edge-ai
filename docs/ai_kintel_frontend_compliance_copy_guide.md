# AI KINTEL Frontend Compliance Copy Guide

## Status

- Stage: 11F - frontend compliance copy guide.
- This is documentation only.
- It does not add frontend code, UI/CSS, React components, route registration, sidebar navigation, backend code, runtime tRPC procedures, provider calls, dependencies, source adapters, source activation, or local RC changes.

## Required Copy

Use these exact short strings where possible:

- `Research-only module. Not investment advice.`
- `WATCHLIST means further manual review only.`
- `Missing data requires manual verification.`
- `Review status does not change scanner label or score.`
- `Paid/deferred source is not configured yet.`

## Forbidden Words As CTA Or Function

Do not use these words as product actions, button labels, feature names, or automated decision outputs:

- `Buy`
- `Sell`
- `Entry`
- `Signal`
- `Recommendation`

These terms may appear only in compliance documentation when describing what must be avoided.

## Preferred CTA Copy

- `View details`
- `Open research details`
- `Review context`
- `Check source status`
- `View methodology`

## Placement

Compliance copy should appear in:

- Header small note.
- Project detail.
- Watchlist / Research Queue.
- Risk alert detail.
- Report/export area if added later.
- Methodology panel.
- Empty/stale/partial data states when manual verification is needed.

## Tone

- Clear.
- Not alarmist.
- Not a legal wall of text.
- Visible but not dominant.
- Specific about data limits.
- Consistent with AI KINTEL dark app UI.

## Copy Rules

- `WATCHLIST` copy must always mean further manual review only.
- Missing data copy must not imply positive assessment.
- Review status copy must stay separate from scanner labels and scores.
- Source copy should distinguish disabled/deferred/not configured from real runtime failures.
- Paid/deferred source copy should not imply that a vendor integration is broken.
- Avoid trader-instruction framing in page headings, empty states, buttons, and detail actions.
